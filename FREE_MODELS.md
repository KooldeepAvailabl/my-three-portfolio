# 🎨 Free 3D Models - Quick Reference

## 🏆 Best Sources for Portfolio Models

### 1. **Sketchfab** ⭐ RECOMMENDED
**URL**: https://sketchfab.com/3d-models?features=downloadable&sort_by=-likeCount

**Best For**: Realistic product showcases (laptops, desks, tech)

**How to Download**:
1. Search for model (e.g., "macbook pro")
2. Filter: "Downloadable" + "Free"
3. Click model → Download → Select "glTF"
4. Extract and use `.glb` file

**Recommended Searches**:
- "macbook pro low poly"
- "developer desk"
- "workspace minimal"
- "laptop computer"
- "coffee cup desk"

---

### 2. **Poly Pizza** ⭐ EASIEST
**URL**: https://poly.pizza

**Best For**: Low-poly, stylized objects

**How to Download**:
1. Browse or search
2. Click model
3. Download GLB directly
4. No account needed!

**Recommended Models**:
- Computer/Laptop
- Coffee mug
- Desk items
- Abstract shapes

---

### 3. **Quaternius**
**URL**: https://quaternius.com/packs.html

**Best For**: Ultimate collections, game-ready

**How to Download**:
1. Download entire pack (ZIP)
2. Extract and find `.glb` files
3. Free for commercial use

**Best Packs**:
- Ultimate Low Poly Office
- Ultimate Modular Sci-Fi
- Ultimate Low Poly Tech

---

### 4. **Kenney Assets**
**URL**: https://kenney.nl/assets?q=3d

**Best For**: Clean, simple designs

**How to Download**:
1. Browse 3D assets
2. Download pack
3. Convert OBJ to GLB if needed

---

### 5. **CGTrader Free**

**Filter**: Free + glTF format

## ✅ Model Checklist

Before using a model:

- [ ] File format: `.glb` or `.gltf`
- [ ] File size: < 5MB (for web)
- [ ] Polygon count: < 50k triangles
- [ ] License: Free for commercial use
- [ ] Textures: Embedded in GLB

---

## 🎨 Model Style Guide

**For Professional Portfolio:**
- ✅ Minimalist, clean designs
- ✅ Low-poly or stylized
- ✅ Neutral colors (can recolor in code)
- ✅ Single object focus
- ❌ Too detailed/realistic
- ❌ Cartoon/game characters
- ❌ Multiple objects (unless scene)

## 🚀 Quick Start

1. Go to https://poly.pizza
2. Search "laptop" or "desk"
3. Download GLB file
4. Place in `/public/models/`
5. Update code in `main.js`
6. Test and adjust scale

---

## 💡 Pro Tips

- **Start simple**: Use low-poly models first
- **Test performance**: Check FPS after adding model
- **Optimize size**: Compress models if > 2MB
- **Match style**: Keep consistent with your design
- **Credit creators**: Add attribution if required

---

## 📱 Mobile Optimization

For mobile devices:
- Use models < 1MB
- Reduce polygon count
- Disable complex animations
- Test on actual devices
